## Changes

### Vote Page (vote.js/wxml/wxss)
- Three-state cycle tap: unselected(0) -> available(2) -> reluctant(1) -> unselected
- First-use guide overlay with animation demo
- Progress bar shows available + reluctant counts separately
- Submit sends slot score object instead of array

### Result Page (result/result.wxml/wxss)
- Top 3 recommendation cards with satisfaction distribution bars
- Consensus level indicator (high/medium/low)
- Preference-weighted heatmap (score-based coloring, not headcount)
- Pareto optimal markers (gold star)
- Slot detail modal shows available/reluctant/unavailable groups

### Cloud Functions
- getEventResult: weighted aggregation algorithm
  - Score = H * (W + alpha*P) where H=hard count, W=weighted sum, P=preference concentration
  - Satisfaction rate, consensus level, Pareto optimal set
  - Paginated response query (up to 100 per batch)
- submitResponse: accepts both old array and new object slot formats
  - Normalizes old arrays to score=2 automatically

### Data Compatibility
- normalizeSlots handles both array and object formats
- Old data treated as all available (score=2)
- Zero migration required
