# Party Time App — 设计改进简报

## 当前状态
微信小程序"聚会时间统计"，用户创建活动 → 其他人在时间格子上勾选"有空" → 系统数人头推荐最佳时间。

## 核心问题：阿罗不可能定理
当前使用简单多数决投票（数人头），触犯 Arrow's Impossibility Theorem 的多个条件：
1. 用户只能二元选择（有空/没空），无法表达偏好强度
2. violates IIA — 增加选项可能改变已有排名
3. 无传递性保证
4. 所有人权重相同，不区分硬约束和软偏好

## 改进目标
1. 支持偏好表达（不只是有空/没空）
2. 引入更合理的投票汇总算法（Borda Count / Condorcet / 加权评分等）
3. 结果展示更丰富（满意度分数、最大共识时段、帕累托最优集合）
4. 保持小程序用户体验简洁（不能因为加了偏好表达就变得复杂难用）

## 技术约束
- 微信小程序 + 云开发（现有架构）
- 需要向后兼容已有数据
- 前端使用 WXML/WXSS/JS（不是 uni-app 或 Taro）
